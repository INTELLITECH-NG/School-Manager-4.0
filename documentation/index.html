Introduction

EZ SchoolMate, it's a complete automation process for managing any school. Our system is web based but it also has a normal desktop application like interface and features. Every work and event that performed manually now, can be performed automatically through our system. Like keeping all records of students and teachers, online admission, result processing, library, send notification to teachers, students or stuffs and many many more. Our system is fairly simple and easy to operate. You don’t need any kind of technical knowledge to operate the system but must have to read this documentation very well.

How to install?

Server Configuration
  • PHP v5.4 to 7 & mysql v5.x
  • Make sure curl is enabled, mbstring is enabled, safe_mode is off, open_base_dir has no value,  allow_url_open is open & mysqli support is on
  • Make sure that the following folders/files are not write protected:
    1. Folder : application/config (777)
    2. Folder : application/core (777)
    3. Folder : download (777)
    4. File : application/install.txt (777)

[Note: If you get "404 page not found" error then make AllowOverride in your virual host and check if .htaccess file is uploaded successfully in root or not]

Installation Manual
  1. Download .zip package
  2. Upload it to your server
  3. Extract the package
  4. Run the uploaded url via browser (http://yourdomain.com/schoolmate/) and you will find a interface to provide the settings for SchoolMate:
    • Hostname : database host name / IP
    • Database Name : create a mysql database for SchoolMate on your host and write that name here
    • Database username  : username of the created database
    • Database password : password of the created database
    • SchoolMate Admin Panel Login Username: this will be used to login as admin
    • SchoolMate Admin Panel Login Password : password to log in as admin
    • Language
    • Institute Name and Institute Address
    • Institute Phone /  Mobile
  5. Click Install SchoolMate Now Button
  6. You are done. Log in with your admin username and password and start using SchoolMate.

Overview

Our system is divided into 4 sections :  admin, teacher, student/guardian and applicant. Admin is the root user or top privileged user. Admin can add different level of sub admins for easy management of the software.

Next section is teacher section. Here a teacher can login and view profile, edit profile, see own class schedule, upload course materials etc. Teacher can get the notification from admin.

In student/guardian section, a student can login and view profile, edit profile, see class routine,,download his course material, read library books etc. Student can also send and receive email for any query and complain through internal messaging system.

Applicants apply online to be admitted and become student.

Admin Panel

Admin panel consists of several settings. These settings are needed to start the system. Admin can do everything on system, like add, edit and delete, send notification to students and teachers etc. Admin can create sub-admin (operator with less permission than admin) and many more. Admin need to be careful because he or she may delete or change critical data that may not meant to be deleted or at least change. So admin is advised to take backup of database on regular basis or before some critical change on system. More on this later.

 

Dashboard:

Dashboard is the home page or index page of admin. Admin will get to see this page right after login. Dashboard is the place for all the statistics of the total system activity. Graphical charts are used for easy understanding. Consider this as the drawing room of your house.

 

Administration:

Administrative settings are sorted as per order of system flow, like you have to define or set roles before creating users. So do not just enter randomly into menus and create add or changes, do it as top down way. More on this later. Settings are very important to understand because the whole application reply on this settings. So, please read the settings manual very carefully.

 

General Settings:

By clicking general settigs link, admin will get the page for initial settings or configuration. Compulsary fields are marked with *. Here admin can set institute name, address, mobile, email, logo, favicon, language.

This is system's sms configuration for sending sms to students, teacher and staffs. SMS are also used in online admission to confirm admission. Without configuring this setting admin cannot send sms. So config it first.

If you forget to enter any compulsary data an form validation error will be shown. Correct your error and resubmit your form. And last but not the least always feel free to contact us in case of confusion and query about the system.

 

Email Settings:

Financial Year:

Financial year is the passed or current year which consists money transaction either by electronicaly or manually happened in school or educational institute. You have to add a financial year every year.

It is different from academic year/session. A financial year most likely to start from the middle of a calender year and its related to money transaction.

On the other hand an academic year likely to start from the begining of the calender year and related to eduacation and addmission of the students. Since financial is related to money transaction admin needs to configure it before performing any money transaction and money transaction related activity like online addmission etc.

This setting is very much flexible i.e. you can enter previous years data before this software implementation. Note that only one financial year is activated at a time. If any other financial year you activate then the currently active year deactivated autometicaly.

Enough of explaination, lets see whats in settings. The landing page of this setting is list of the financial years. It contains year list, their status (active or inactive), and action (update). And also search facility by financial year. Just enter keyword for financial year and hit search button and your desired search will come. Admin can update a financial year by clicking pencil button under action tab and add a year by clicking add button on top left of the page. And there is no delete button. Add and Edit functions are same as you have encountered previously.

 

Session/ Academic Year:

Session is the academic year. It related to all the educational information about student like exam, result, promotion, addmission etc. So academic year must be configured before using all the student related activity like addmission, promotion etc. Functionality is same as financial year. Only difference is there is no active and inactive options available. Many session can be activated at the same time.

 

Class management:

Class management consists the structure of defining class, period, section, shift and group. Where defining class and period is mandatory and section,  shift & group.deperatment are optional. Without adding classes (exaple: class-I, class-II, class-III) no student related feature will work. On the other hand, periods are  needed to assign class to teachers as well as build routine and without assigning class result processing will not work.

Add, edit and search functionality is available same as before. click add button to add and click pencil button to edit and enter string in search field select search criteria and hit search.

 

Period/Class Period: .

Without class period is out of context. So admin has to add or define period after class. For example period names can be period-1, period-2, period-3 etc.  Period are needed to populate class routine. Add edit and search functionality is same as before.

 

Section:

Sections are the divisions of class due to large some of student. Educational Institute may have section or not. So we made it optional. As for example section name can be section-A, section-B, section-C etc. Add, edit and search available.

 

Shift: Shift is the division of time taking class. Like morning shift, day shift, night shift etc. This is also optional. Functionality is same as section.

 

Group or Department:

Group or department are the groups of subject like science, arts , commerce etc . An educational institute may have ( university, college) or may not have (primary School) group or department. So we made it optional. Functionality is same as shift or section.

 

Course management:

Course management is the managing course for different classes. You need to define class and session before adding course.

You have to define course for each group/dept and  for each session. Yes, know it's tedious job to add couse for each group/dept and for each session. We have yearly settings to copy data of previous seeion or financial year. But you have to add data at the first place. Add, edit and search facility are also available.

Click add button on course list page. Add page form will come. Provide all the necessary information, select class, group and session form dropdown and click save. You will see success message on if the data inserted to database successfully. Credit/Marks will be used in result processing.

 

Online admission configuration:

This is the online admission facility for students. Admin can enroll students completely automatically with help of this facility. Session, class and group or department need to be predefined before online admission. Search, edit and add are available.

Applicant can only apply any of "open" admission. You can set  many properties like application last date, admission last date, admission test information etc. You can also choose if sms will be send or not to the applicants. You can close any admission if you want.

Online application and admission process is described under applicant panel in details, this is only settings here.

 

Result management:

Result management is the settings for all exam and  exam marks. There are four submenus. Thet are exam, gpa, cgpa and result.

Exam: Exams are list of exam for different classes. Class, session and group or department needed to be defined previously to set exam.

There is a field called status. Status means the current of state of exam. Status of exam remians incomplete until exam paper checked, marks given and result processed (discussed in admin panel> result section). Teachers will login to their panels and input obtained marks of students (discussed i teacher panel). If status is given complete then no marks giving and result processing is possible. You can not delete an exam.

When you create an exam there is a field called result type (marks/gpa/cgpa) which specify the result processing formula.

 

 GPA configuration:

GPA stands for grade point average. GPA configuration is the distribution of grade point according to marks gained in subjects. Its scale is 5. Note that in distribution of grade points there must not have any gap or overlapping. Look carefully about our demo gpa config. It's distribution in short is 0-39, 40-44, 45-49, 50-54, 55-59, 60-64, 65-69, 70-74, 75-79, 80-100. Observe that there is no gap or overlapping. If you leave any gap ( 0-39, 41-45) or overlapping (0-40, 40-45) you will see some unexpected result. And in the consequence the whole result calculation will collapse. So keep an eye on this.

 

 

CGPA Configuration:

CGPA Stands for cumulative grade point average. It's scale is 4. All the operation and handling is same as gpa configuration. In CGPA processing, obtained grades & course credits are considered.

 

Optional Course:

Some schools may have an optional course. So we set up an special procedure to count this. Marks or grade for this optional course most likely to be calculated partialy. Like if the grade point is 5 or marks more than or equal to 80 obtained by the student then the grade point counts as only 5 - 2 = 3. i.e. whatever student's obtained grade is 2 will be subtracted from this and negative grade will not be considered. This grade will be added to his total obtained grade point and divided by total number of subjects without optional course.

 

Role:

Role means assigning the ability of which user can do what in this system. There can be one or more user of a single role, like student is a roll and there are a lot of student in an educational institute. Admin, teacher, and student are the default roles i.e. they are created implicitly.

User with admin role can do anything including adding other roles of different permissions or disable a role. Since role is the container of users so admin has to insert roles other than teacher and student like librarian, guard before inserting user of these roles etc.

There are several types of permission like read, write, add, delete and approve for a role. Like the role of librarian should have all permission or privilages for operating and maintaining the library. In the snapshot below you can see the available roles on this system. There is also add button to add a new role, details button (magnifying glass) to view role details and edit button (pencil icon) to edit an existing role and search feature.

Add role is the role adding facility. Click add button in the role page, a form will come. Enter role name, status (active or inactive) and check suitable privilages according to role and hit save. You will get confirmation on success.

Edit role is same as add role. Only difference is that the privilages given to a role is selected in edit form for remembering. Make necessary changes in existing settings of role and press save.

Role details is the list of privilages a roll holds. Click magnifying glass button to see the details of a role.

 

User:

User panel is the list of all users who belongs to educational system like student, teacher and employee. The admin, student and teacher roles are crated in default. So user with these roles inserted in database autometically.

Users of other roles (e.g. librarian, guard etc) have to be created manualy. Since role contains user (i.e. role student, username John Doe) admin has to create role first. User panel contains list of all users with username (eg. David, Peters) , role name (list of permission provided to individual or group eg. Admin, teacher, student), user type (eg. Admin as operator, student as individual created in default) reference no., status (active or deactive) and action button.

Under action button there is update button. Admin can add a new user by clicking add button. Admin can activate or deactivate a user, set his user type etc. all this can be done by clicking pencil button under action tab.

Look at the add user form. There are some fileds and all the fields are required. The first field is username which is the username for holding a role defined previously. Password is the password for acount of this user. confirm password is giving the same password. Role is the dropdown list of roles previously defined. Status of this user (active or deactive according to current state of this post in educational institute), user type (operator if sub admin or another admin or individual if employee or staff), individual type ( teacher, student, employee created in default) reference id ( identification number given manualy or autometicaly when enlist into enducational institiute). Fill the the form with suitable data and hit save. You will get a confirmation on success.

Click pencil button to edit user at the end of a row of any user under action column. Edit user is same as add user. Just the fields are selected or filled with previously given values. Note that username cannot be changed. Make necessary changes and hit save. You will get confirmation on success.

 

Account management:

Account management is account management facility of the accounts that an educational institute keeps for money management and transaction. This menu has four sub menus and they are account type, account head, pay slip and payment method.

 

Account type:

Account types is the list of account types that a educational institute holds. Like charity, govt, non-govt etc. Note that account deals with money and for money transaction time is important so admin has to define financial year first and session. Click add button to add a new account, just enter account name and hit save. Click pencil button to edit an account. You cannot delete any account.

 

Account heads: accounts heads are sub division of the account types. E.G. charity may contains account heads for needy students, poor employee etc. Since account types contain account heads so admin has to define account type first. To add an account head click add provide suitable values and hit save. To edit hit pencil icon, make necessary changes and hit save.

 

Pay slip: Pay slip is the details report of money transaction under any slip type. Class, department, session, pay slip all are needed to be predefined before creating pay slip. On this page one can see slip name, slip type, class, group, session, amount etc. To add a pay slip click add button, to edit click pencil button. Playslip is needed in admission/online admission.

The add page is created autometicaly accoding to account type, account heads and slip types. Admin should fill this form with data according to educational institute rules. Fields for sum of money is optional. But admin must provide slip name, class, group / department, session and slip type.

Edit pay slip is same as add. Just the values given previously selected or check. Make necessary changes and hit save.
